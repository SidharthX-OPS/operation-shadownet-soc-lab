# ALERT-HIGH-T1053-Scheduled-Task

| Field | Value |
|---|---|
| Rule ID | DET-004 |
| Severity | HIGH |
| MITRE Technique | [T1053.005 — Scheduled Task/Job: Scheduled Task](https://attack.mitre.org/techniques/T1053/005/) |
| Data Source | Windows Security EID 4698 (Scheduled task was created) |
| Index | wineventlog |
| Validation Status | PASS |

---

## Description

Detects scheduled task creation via `schtasks.exe`, `at.exe`, or the Task Scheduler COM API. Security EID 4698 fires regardless of the creation mechanism and includes the full task XML, which contains the embedded command line — making this one of the most evidence-rich events in the Windows log.

## SPL

```spl
index=wineventlog source="WinEventLog:Security" EventCode=4698
| eval severity="HIGH"
| table _time, ComputerName, SubjectUserName, TaskName, TaskContent
| sort - _time
```

## Validation Evidence

Atomic Red Team T1053.005 #1 created task `T1053_005_OnStartup` under `NEXCORE-WIN\WIN`. The embedded XML contained:

```xml
<RegistrationInfo>
  <Date>2026-05-23T00:46:47</Date>
  <Author>NEXCORE-WIN\WIN</Author>
  <URI>\T1053_005_OnStartup</URI>
</RegistrationInfo>
<Triggers>
  <BootTrigger>
    <Enabled>true</Enabled>
  </BootTrigger>
</Triggers>
<Actions Context="Author">
  <Exec>
    <Command>cmd.exe</Command>
    <Arguments>/c calc.exe</Arguments>
  </Exec>
</Actions>
```

A `BootTrigger` set to run `cmd.exe /c calc.exe` on system startup — a textbook persistence pattern. The full `TaskContent` field is preserved in the Splunk event, which is exactly what an IR analyst needs for triage.

## False Positive Notes

EID 4698 is generated by every scheduled task creation including:
- Windows Update creating maintenance tasks
- Software installers (Office, Edge, Adobe) creating updater tasks
- IT-deployed agents (EDR, RMM) registering health checks

**Tuning approach:** Whitelist `SubjectUserName=SYSTEM` for OS-generated tasks, and known software vendor patterns in `TaskName` (e.g., `\Microsoft\Windows\*`, `\Mozilla\*`, `\Adobe\*`). Anything in the root namespace (`\TaskName` with no parent folder) is unusually suspicious — that's the schtasks.exe default and rarely how legitimate software registers tasks.
